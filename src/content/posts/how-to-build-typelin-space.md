## 緣起：為什麼選擇 Fuwari？

在眾多靜態網站生成器中，我最終選擇了 **Astro** 作為技術棧核心。

Astro 的 **Island Architecture（孤島架構）** 讓我能夠在保持頁面輕量的同時，按需載入互動元件。這對於部落格這類以內容為主的網站來說，是近乎完美的解決方案。

本站的基礎架構來自兩個重要的開源專案：

| 專案 | 貢獻 |
|------|------|
| [saicaca/fuwari](https://github.com/saicaca/fuwari) | 原始 Astro 部落格模板，提供了極簡美學的設計基礎 |
| [afoim/fuwari](https://github.com/afoim/fuwari) | 繁體中文化版本，提供了在地化的介面文字與部分功能優化 |

在這兩個版本的基礎上，我進行了大量的客製化改造，打造出現在這個獨一無二的技術空間。

---

## 技術架構概覽

整體架構採用現代 JAMstack 理念，將前端、後端、部署三者完全解耦：

| 層級 | 技術選型 | 說明 |
|------|---------|------|
| **前端框架** | Astro 5.x | 支援 SSG/SSR 混合模式，零 JS 預設載入 |
| **UI 框架** | Svelte + Tailwind CSS | 元件化開發、原子化樣式 |
| **內容管理** | Markdown + Content Collections | 類型安全的內容層 |
| **部署平台** | Cloudflare Pages | 全球邊緣網絡，自動 CI/CD |
| **資料儲存** | Cloudflare D1 (SQLite) | Serverless 關聯式資料庫 |
| **運算層** | Cloudflare Workers | 邊緣運算，毫秒級冷啟動 |

## 主要改造內容

### 1. 視覺與品牌重塑

- **個性化 Logo**：設計了專屬的幾何圖形標誌，融合品牌識別
- **配色系統**：基於 oklch 色彩空間進行動態主題色調整
- **頁面重構**：重新設計了 404 錯誤頁、關於頁等核心頁面
- **動態背景**：整合第三方 API 實現每次訪問的隨機圖片背景

### 2. 邊緣計數系統開發

放棄了傳統的第三方統計方案（如 Google Analytics、百度統計），自行基於 Cloudflare 全家桶開發了輕量級計數系統：

**核心元件：**

| 元件 | 功能 |
|------|------|
| **Worker Script** | 接收統計請求、處理業務邏輯 |
| **D1 Database** | 儲存頁面路徑、瀏覽次數、訪客數 |
| **前端元件** | 展示計數、防作弊機制 |

**技術亮點：**

- **全球邊緣部署**：利用 Cloudflare 的 300+ 邊緣節點，確保全球訪問低延遲
- **智能去重**：透過 localStorage 記錄訪問時間戳，24 小時內同一訪客不重複計入
- **無 Cookie 設計**：完全基於客戶端儲存，符合 GDPR 等隱私法規精神
- **容錯機制**：網路異常時靜默失敗，不影響頁面瀏覽體驗

### 3. 構建與部署優化

- **增量式構建**：利用 Cloudflare Pages 的 Git 整合，推送即部署
- **資源壓縮**：自動化圖片優化、CSS/JS 壓縮
- **快取策略**：靜態資源長期快取，HTML 頁面即時更新
- **Preview Deployments**：每個 PR 自動生成預覽環境

### 4. 其他功能整合

- **側邊欄強化**：動態分類、標籤雲、社交連結
- **搜尋功能**：基於 Pagefind 的全站靜態搜尋
- **RSS 訂閱**：自動生成 RSS 2.0 feed
- **Sitemap**：動態生成 XML Sitemap，提升 SEO
